About WebHistory - by Noak Palander

WebHistory is a website designed to teach the user the most important things about the web we use on daily basis.

It's developed using HTML, CSS, Javascript, and python/flask as it's backend.


  • - Week 1

    During the first week I setup the basic html code (index.html)

    The code found in index.html is the home page, I setup a header "menu" with animations and links to the about page (this) and back to the home page.

    To get the animation and style up I used css and chose to go for a dark purple-theme, utilizing some nice blue/purple color schemes. Including some usage of css-gradients.

    The first week also consited of me fetching some information that would suit the website's purpose.

  • - Week 2

    The second week of the project was mostly aimed at getting the site to look better, in order for this to work I mostly coded in css.

    I also had to get some free images that would look good on the site, these are displayed on the home page and on the history page.

    The goal that I achieved during this week was for the website to get it's look and layout fixed.

  • - Week 3

    The third week was mostly dedicated to coding in client-side javascript. It had to be used to dynamically load the text and images needed on the history page.

    This could have been excluded if I opted to "hard-code" the text on the site instead of dynamically load them.

    The latter seemed like a better choice as then I could more easily change the text on the site without having to change the layout and the html code

    Loading texts and images by pure client-side javascript wasn't really possible, so I had to setup a backend server.

  • - Week 4, the final week

    The final week of the project was mostly server development and last touches on the site.

    I started developing a backend server using Node.js, but it was later replaced by Python's Flask module.

    I'd rather setup the server using C++'s Boost::Asio framework, however the time-limit made it a bad decision and was therefore not used.

    I mostly chose Flask over Node due to my lack of previous experiences with Javascript compared to Python both of the frameworks would probably suffice.

    The server was setup on a linux machine running a debian based OS. And a port was forwarded for the server

    I also restructured the file and folder structure for the project, having a "public" folder for all files supposed to be avalible on the site.
